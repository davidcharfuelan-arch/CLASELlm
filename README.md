y# Sistema de Pedidos a Domicilio

Este proyecto representa una aplicación similar a Rappi que permite a los usuarios solicitar productos o comida desde diferentes establecimientos y recibirlos en su domicilio.

## Requerimientos Funcionales

1. El sistema debe permitir que los usuarios creen una cuenta.
2. El sistema debe permitir que los usuarios busquen restaurantes o tiendas.
3. El sistema debe permitir agregar productos al carrito de compras.
4. El sistema debe permitir realizar pedidos.
5. El sistema debe permitir hacer seguimiento del pedido en tiempo real.
6. El sistema debe permitir realizar pagos en línea.

## Requerimientos No Funcionales

1. El sistema debe procesar los pedidos en menos de 5 segundos.
2. El sistema debe garantizar la seguridad de las transacciones.
3. El sistema debe estar disponible las 24 horas del día.
4. El sistema debe soportar miles de usuarios al mismo tiempo.
5. El sistema debe tener una interfaz intuitiva y fácil de usar.
6. El sistema debe proteger la información personal de los usuarios.
